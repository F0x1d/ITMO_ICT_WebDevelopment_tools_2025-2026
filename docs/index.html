Лабораторная работа 1

Тема: разработка веб-приложения для поиска партнеров в путешествие.

Приложение позволяет регистрировать пользователей, авторизовывать их через JWT, создавать поездки, добавлять остановки маршрута, связывать пользователей с интересами и управлять участниками поездок.

from app.core.config import settings

engine = create_engine(settings.database_url, echo=True)


def get_session():
    with Session(engine) as session:
        yield session

Модели данных

Всего используется 6 таблиц.

Таблица Назначение Ключевые поля
users Пользователи системы email, username, travel_style, password_hash
interests Справочник интересов name, description
trips Поездки пользователей title, departure_city, destination_city, organizer_id
trip_stops Остановки внутри маршрута trip_id, location, days
user_interests Связь пользователя и интереса user_id, interest_id, experience_level
trip_participants Связь пользователя и поездки trip_id, user_id, status, message, contact_preference

Связи

  • users - trips: one-to-many
  • trips - trip_stops: one-to-many
  • users - interests: many-to-many через user_interests
  • users - trips: many-to-many через trip_participants

Ассоциативные сущности имеют собственные поля связи:

  • user_interests.experience_level
  • trip_participants.status
  • trip_participants.message
  • trip_participants.contact_preference

Авторизация и безопасность

  • Регистрация: POST /auth/register
  • Логин: POST /auth/login
  • JWT access token
  • Аутентификация через Bearer token
  • Хэширование паролей
  • Смена пароля: POST /users/me/change-password

В проекте настроена система миграций Alembic, ревизии создаются и применяются через стандартные команды alembic revision --autogenerate и alembic upgrade head.

Реализованные эндпоинты

Auth

  • POST /auth/register
  • POST /auth/login

Users

  • GET /users
  • GET /users/me
  • GET /users/{user_id}
  • PATCH /users/me
  • POST /users/me/change-password
  • DELETE /users/me
  • POST /users/me/interests
  • PATCH /users/me/interests/{interest_id}
  • DELETE /users/me/interests/{interest_id}

Interests

  • GET /interests
  • GET /interests/{interest_id}
  • POST /interests
  • PATCH /interests/{interest_id}
  • DELETE /interests/{interest_id}

Trips

  • GET /trips
  • GET /trips/{trip_id}
  • POST /trips
  • PATCH /trips/{trip_id}
  • DELETE /trips/{trip_id}
  • POST /trips/{trip_id}/stops
  • PATCH /trips/stops/{stop_id}
  • DELETE /trips/stops/{stop_id}
  • POST /trips/{trip_id}/participants
  • PATCH /trips/participants/{participant_id}
  • DELETE /trips/participants/{participant_id}

Вложенные GET-запросы

  • GET /users/{user_id} и GET /users/me возвращают пользователя, его интересы, созданные поездки и поездки участия.
  • GET /trips/{trip_id} возвращает поездку, организатора, остановки и участников.
